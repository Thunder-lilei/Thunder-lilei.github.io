Spring 事务
2020-12-08
spring 事务
technology

作者 ： 我有一把小榔头丶

概念

  • 数据库操作最基本单元，逻辑上一组操作
  • 整租成功或失败

  • 特性ACID

    1. 原子性
      过程不可分割
    2. 一致性
      总量不变
    3. 隔离性
      多事务之间不影响
    4. 持久性
      提交后表中数据变化

环境

  1. 事务中注入连接池
  2. dao层实现类
    注入事务对象

事务管理

  • 事务添加到Service层（业务逻辑层）
  • 操作
    1. 编程式事务管理
      臃肿
    2. 声明式事务管理
      一般用这种
      底层使用AOP管理

  1. 配置事务管理器

    JDBC模板的管理器
    datasource就是上面的连接池

  2. 开启事务注解

声明式事务参数管理

  1. propagation事务传播行为

    事务方法：对数据库数据进行变化的行为

  2. 事务隔离级别

    有特性成为隔离性， 多事务之间不会产生影响
    不考虑隔离性可能产生的问题：

    1. 脏读

      一个未提交事务读取到另一个未提交事务

    2. 不可重复读（不算问题 算现象）

      一个未提交事务读取到另一个提交事务修改数据

    3. 虚（幻）读

      一个未提交事务读取到另一个提交事务添加数据

    4. 隔离级别

      mysql默认第三个

  3. timeout超过时间

    事务超时不提交
    默认值 -1 单位秒

  4. readyOnly 只读

    查询操作
    默认值 false 可以查询 也可以增删改，设置为true 只读

  5. rollbackFor

    设置哪些异常回滚

  6. norollbackFor

    设置那些异常不回滚
