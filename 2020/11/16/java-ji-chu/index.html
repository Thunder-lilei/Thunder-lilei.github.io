Java机制

  • jvm（Java虚拟机）

    • 运用硬件或软件手段实现虚拟的计算机
    • 有自己完善的硬件架构
    • 可运行Java字节码
    • 有一个解释器组件，可以实现Java字节码和计算机操作系统之间的通信

  • 垃圾回收机制

    • C/C++需要析构函数回收
    • 不再使用的内存区自动回收
    • 对象回收需要移除其所有引用
    • 可能出现内存溢出

Java应用程序

  • Java严格区分大小写
  • class表示类，Java应用程序全部内容都必须在类中
  • 源代码文件名必须与共有类名相同
  • main函数，共有的public，静态的static，无返回类型void，带有一个字符串数组类型的参数String args[]
  • 每个Java应用程序都必须有一个main函数
  • 一对花括号{}表示程序的开始和结束
  • 编写第一个应用程序
    • 记得目录名
    • Javac 编译文件后出现 HelloWorld.class文件

数据类型

  • 8种基本数据类型

    • 整型：byte（-128——127） short（-32768——32767） int（-231——230） long

      • 1，2，4，8个字节

    • 浮点型：float double

      • 4，8个字节
      • float后缀f，没有默认为double

    • 字符型：char（0——65535）

      • 2个字节
      • 不能将字符串字面量分配给char变量，即使String字面量只有一个字符。
      • Java汉字只有一个字节，需要单引号
      • 字符转义序列
      • 字符字面量可以表示为“\ uxxxx”形式的Unicode转义序列（ASCII字符集占据Unicode字符集中的前127个值）

    • 布尔型：boolean

      • false true

    • 下划线数字字面量

      • 可以在两位数字之间使用下划线
      • 允许以八进制，十六进制和二进制格式使用下划线

  • 引用数据类型

    • 字符串
    • 数组
    • 自定义

  • 强类型

    • Java是强类型语言
    • 每个变量、表达式都有类型，每个类型都有严格定义
    • 检查所有分配的类型的兼容性
    • 没有自动强制或有冲突类型的转换

变量

  • 标识符

    • 赋予变量、类和方法的名称
    • 字母、下划线、美元符号$开头
    • 大小写敏感
    • 不能是关键字

  • 声明

    • 数据类型 变量名称

  • 自动类型转换

    • 小到大 自动类型转换
    • 大到小 强制类型转换（目标类型 类型名=（目标类型）值）
      会导致数据丢失
    • 数字型、字符型、布尔型相互不兼容

程序结构

  • 顺序结构

    按出现的顺序依次执行

  • 分支结构

    根据条件判断的结果不同，执行不同功能

  • 循环结构

    重复执行一段代码

  • 结束

    • break跳出整个循环
    • continue跳出当前循环
